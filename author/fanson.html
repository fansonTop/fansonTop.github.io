java面向对象

初识对象

  1. 什么是对象：对象就是东西（物件），万物界对象
  2. 对象的构成：
    1. 属性：数据特征
    2. 方法：操作，行为
  3. 显示世界中现有对象，对象多了就有了分类

java中的类和对象

  1. java中现有类型，后有对象
  2. 在java中定义类
    1. 在类中定义这个类的对象有哪些属性和方法
    2. 声明属性
    3. 声明方法
  3. 在java中实例化类的对象
    1. 一个类可以实例化无数对象
    2. 对象时程序操作的主体
    3. 通过对象名打.来访问对象的属性，来调用对象的方法
    4. 实例化对象的格式

      <类名> <对象名> = new <类名> ();
      Student student = new Student();

对象属性的默认值

  1. 对象的属性是有默认值的
  2. 基本数据类型的默认值为(0/0.0/false)
  3. 引用类型的数据默认值为null

方法重载

  1. 方法签名
    1. 每个方法的定义包含：修饰词，返回类型，方法名，参数列表
    2. 方法签名=方法名+参数列表
    3. 每个类中不允许方法签名完全相同的方法
    4. 但是方法名相同，参数列表不同时允许的
  2. 什么是方法重载：在同一个类中方法名相同，参数列表不同的多个方法
    1. 参数列表不同，包含：数量/类型/顺序
    2. 方法的秀师傅和返回值类型对方法重载时无关的
  3. 方法重载在代码编译时确定，运行哪个方法，又称为编译期绑定
  4. 系统中的方法重载和方法重载的好处
    1. 方法重载的好处在于不需要程序员记忆过多功能相抵的方法名，只需要通过传入的参数区分即可
    2. 系统中利用方法重载编写了很多内容，供程序员使用

构造方法

  1. 什么是构造方法：在对象被实例化运行时的一个特殊方法
  2. 为什么需要构造方法：一般在构造方法中赋予属性初值
  3. 怎么编写一个构造方法：
    1. 构造方法没有返回值类型
    2. 构造方法的方法名必须和类名相同
    3. 在实例化对象时，会自动运行构造方法
    public <类名>() {
    // 构造方法
}
  4. 构造方法的注意事项
    1. 如果一个类中没有编写构造方法，系统会自动给这个类分配一个默认的无参构造
    2. 如果一个类中编写了任意一个构造方法，那么系统就不会给这个类分配默认构造方法了
    3. 构造方法可以重载
    4. 构造方法只能在实例化对象时调用，不能使用对象打.调用

成员变量和局部变量

  1. 类中定义的属性就是成员变量
    方法中定义的变量和方法的形式参数就是局部变量
  2. 区别
    1. 作用域不同
      1. 成员变量：类中所有方法均可以使用
      2. 局部变量：只有定义它的方法中可以使用
    2. 初始默认值不同
      1. 成员变量：0、0.0、false、null
      2. 局部变量：没有初始值
    3. 优先级不同
      1. 如果在一个方法中使用了成员变量和局部变量同时存在的变量名，优先使用局部变量

this关键字

  1. 什么是this：方法中对象对自身的引用
  2. this.属性：能区分成员变量和局部变量
  3. this.方法：调用当前类中的方法
  4. this(<参数>)：调用当前类的构造方法，必须写在构造方法中，而且必须是第一行

数据类型和引用数据类型

  1. 基本数据类型的8种：基本数据类型变量保存的值在内存的栈中
  2. 引用数据类型

    String，数组，类（自造类）
    引用数据类型变量保存在栈中的值是它引用对象的引用值
